# マッピングボタンの機能と仕組み

本プロジェクト（MOTION_PLAYER MP-5）では、画面上のセンサー入力（タッチ座標および加速度）とオーディオパラメータを動的に結びつける（マッピングする）機能を備えています。
マッピングには以下の3つのモード（GYRO, XY PAD, 16 PAD）が用意されています。

## 1. オーディオパラメータ（操作対象）
アプリのUIには以下の3つのオーディオパラメータが用意されています。音声処理には Web Audio API ではなく `Tone.js` を使用しています。

*   **Pitch (ピッチ)**: 音声のピッチ（音程）を制御します。`Tone.GrainPlayer` の `detune` を操作します。
*   **Filter (フィルター)**: ローパスフィルターのカットオフ周波数を制御します。`Tone.Filter` の `frequency.rampTo()` を使用して滑らかに操作します。
*   **Speed (スピード)**: 音声の再生速度を制御します。`Tone.GrainPlayer` の `playbackRate` を操作します。

## 2. センサー入力とモード（操作元）
画面上のタブで3つのモードを切り替えることができます。

*   **GYRO**: 端末の加速度センサーを使用してパラメータを操作します。各パラメータには以下の4つのいずれかを割り当てることができます。
    *   **NONE**: マッピングを無効にします。
    *   **A_X (Accelerometer X)**: 端末の加速度センサー（X軸方向の傾き）
    *   **A_Y (Accelerometer Y)**: 端末の加速度センサー（Y軸方向の傾き）
    *   **A_Z (Accelerometer Z)**: 端末の加速度センサー（Z軸方向の傾き）
*   **XY PAD**: 画面上のXYパッドをタッチ・ドラッグして操作します。X軸とY軸それぞれに任意のパラメータ（Pitch, Filter, Speed）を割り当てることができます。
*   **16 PAD**: 16個のパッド（4x4）をタップして、離散的なピッチ（半音階）を即座に適用します。

## 3. UIとマッピングの仕組み（GYROモード）
画面の「マトリックス」部分には、3つのパラメータごとに4つのセンサースロット（マッピングボタン）が**横方向（水平）**の行（row）として並んでいます。

1.  **UIからの選択**: ユーザーがいずれかのボタン（スロット）をタップすると、そのスロットの見た目がアクティブ状態に変化します。
2.  **JavaScript上の更新**: タップイベント (`onclick`) によって、内部の `mapping` オブジェクトが更新されます。
    *   例: ピッチの行で `A_Y` を選ぶと、`mapping['pitch'] = 'A_Y'` という形で状態が保存されます。
3.  **リアルタイムな音声への反映**: JavaScriptの `draw()` メソッド（`requestAnimationFrame` で毎フレーム呼び出されるループ）内で、常に現在の `sensors`（加速度）やXYパッドの座標値が読み出されます。
    *   `mapping` 等に設定された状態に基づき、現在の値（0.0 〜 1.0）を取得します。
    *   取得した値を使って、`Tone.js` の各種ノードに値を適用します（例: `filterNode.frequency.rampTo(...)`）。これにより、タッチや端末の傾きがリアルタイムで再生音に反映されます。
