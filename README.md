# マッピングボタンの機能と仕組み

本プロジェクト（MOTION_PLAYER MP-5）では、画面上のセンサー入力（タッチ座標および加速度）とオーディオパラメータを動的に結びつける（マッピングする）機能を備えています。

## 1. オーディオパラメータ（操作対象）
アプリのUIには3つのオーディオパラメータの列が用意されています。

*   **Pitch (ピッチ)**: 音声の再生速度（音程）を制御します。Web Audio APIの `AudioBufferSourceNode` の `playbackRate` を操作します。
*   **Filter (フィルター)**: ローパスフィルターのカットオフ周波数を制御します。Web Audio APIの `BiquadFilterNode` の `frequency` を操作します。
*   **Volume (ボリューム)**: 音量を制御します。Web Audio APIの `GainNode` の `gain` を操作します。

## 2. センサー入力（操作元）
各パラメータには以下の5つのセンサーのいずれかを割り当てることができます。これらの値はすべて 0.0 〜 1.0 の範囲に正規化されます。

*   **T_X (Touch X)**: 画面のタッチ位置またはマウスポインター（X座標・水平方向）
*   **T_Y (Touch Y)**: 画面のタッチ位置またはマウスポインター（Y座標・垂直方向）
*   **A_X (Accelerometer X)**: 端末の加速度センサー（X軸方向の傾き）
*   **A_Y (Accelerometer Y)**: 端末の加速度センサー（Y軸方向の傾き）
*   **A_Z (Accelerometer Z)**: 端末の加速度センサー（Z軸方向の傾き）

## 3. UIとマッピングの仕組み
画面中央の「マトリックス」部分には、3つのパラメータごとに5つのセンサースロット（マッピングボタン）が縦に並んでいます。

1.  **UIからの選択**: ユーザーがいずれかのボタン（スロット）をタップすると、そのスロットの見た目がアクティブ状態に変化します。
2.  **JavaScript上の更新**: タップイベント (`onclick`) によって、内部の `mapping` オブジェクトが更新されます。
    *   例: ピッチの列で `A_Y` を選ぶと、`mapping['pitch'] = 'A_Y'` という形で状態が保存されます。
3.  **リアルタイムな音声への反映**: JavaScriptの `draw()` メソッド（`requestAnimationFrame` で毎フレーム呼び出されるループ）内で、常に現在の `sensors` の値が読み出されます。
    *   `mapping` に設定されたセンサーから現在の値（0.0 〜 1.0）を取得します。（例: `const pVal = sensors[mapping.pitch];`）
    *   取得した値を使って、`setTargetAtTime` メソッドによりWeb Audio APIの各種ノードに値を滑らかに適用します。これにより、タッチや端末の傾きがリアルタイムで再生音に反映されます。
